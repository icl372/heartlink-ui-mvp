## 1. 产品定位

心意链接 / HeartLink 是一个轻量的心意链接生成工具：用户写下一句想表达的话，选择场景、填写信息、生成并编辑文案、选择风格、预览后生成一个可分享链接。接收者打开链接后，像拆开一份小礼物一样经历「封面 → 正文 → 接收完成」的仪式感流程。

核心价值：把普通文字表达包装成一个温柔、私密、适合手机打开的小礼物链接，而不是让用户制作复杂 H5 或设计作品。

## 2. 目标用户

创建者：

1. 想向父母、伴侣、朋友、同事表达感谢、祝福、道歉、鼓励或小心意的人。
2. 想把转账说明、资助说明或一段普通文字变得更有仪式感的人。
3. 不想学习复杂编辑器，只想快速生成一个可分享页面的人。

接收者：

1. 通过聊天工具收到链接的人。
2. 无需登录、无需下载 App，只需打开链接并阅读。

## 3. 用户痛点

1. 普通聊天消息缺少仪式感，容易被聊天记录淹没。
2. 红包或转账体验偏交易，难以承载“我记得你、感谢你、心疼你”的表达。
3. 传统 H5 或贺卡工具过重，模板、排版和样式选择成本高。
4. 用户真正需要的是快速生成一份“可被打开的小惊喜”，不是做复杂设计。

## 4. MVP 范围

MVP 只覆盖当前 heartlink-ui-mvp 已验收 UI 基础已经呈现的能力：

1. 创建端从首页进入，完成场景选择、信息填写、AI 文案生成/编辑、风格选择、预览、生成链接。
2. 接收端打开链接后，完成封面、正文、接收完成三状态体验。
3. 提供原型级 loading、AI 失败、网络错误、复制成功/失败、链接不存在、链接过期状态展示。
4. 不包含真实后端、真实 AI 接口、真实 token 查询、登录、支付、模板市场或复杂编辑器。

## 5. 页面清单

创建端页面：

1. 首页：品牌、定位文案、开始创建入口、接收端预览入口。
2. 场景选择页：选择感谢、祝福、道歉、鼓励、小心意。
3. 信息填写页：填写收信人、署名、想说的话、语气风格；小心意场景下显示金额字段。
4. AI 文案生成页：展示“正在生成文案”的 loading 状态。
5. AI 文案结果页：展示 AI 已生成文案，支持编辑标题、正文、引用、署名；支持重新生成。
6. 风格选择页：选择温柔信纸、复古收据、诗意卡片、简约便签。
7. 预览页：展示接收端封面和正文的组合预览，支持调整风格或生成链接。
8. 生成成功页：展示专属链接、复制按钮、打开预览效果、再创建一份、隐私提示。

接收端页面/状态：

1. 加载中：骨架屏和“正在加载心意…”。
2. 封面：收信人、引导文案、场景图标、开启按钮、HeartLink 标识。
3. 正文：英文标签、标题、正文、引用、署名、日期/场景标识、接收按钮。
4. 接收完成：正文保留，接收按钮变为金色完成态，展示接收时间和珍藏提示。
5. 链接不存在：404 / Not Found、说明文案、查看示例效果按钮。
6. 链接过期：Link Expired、说明文案、查看示例效果按钮。

## 6. 字段清单

创建端输入字段：

1. 场景：感谢、祝福、道歉、鼓励、小心意。
2. 收信人：必填，placeholder 为“例如：最亲爱的妈妈”。
3. 署名：可填，placeholder 为“例如：您的女儿”。
4. 想说的话：必填，建议 30-150 字，显示字数提示。
5. 金额：仅小心意场景显示，仅用于文案展示，数字输入，前缀为 ¥。
6. 语气风格：真诚、温柔、可爱、克制、正式、诗意。
7. AI 生成后可编辑字段：标题、正文、quote / 引用句、按钮文案、署名。
8. 风格：温柔信纸、复古收据、诗意卡片、简约便签。

生成结果字段：

1. 专属链接：原型格式为 `heartlink.app/to/{recipient}-a9f2`。
2. 复制状态：复制成功、复制失败。

接收端展示字段：

1. 收信人。
2. 标题。
3. 正文。
4. 引用。
5. 署名。
6. 场景。
7. 接收日期。
8. 完成态文案。

## 7. 核心流程

创建流程：

1. 首页 → 点击“开始创建心意”。
2. 场景选择 → 选择一个场景 → 点击“下一步：填写信息”。
3. 信息填写 → 填写收信人、署名、想说的话、语气风格；小心意场景可填写金额。
4. 点击“AI 生成专属文案”。
5. 进入“正在生成文案”loading。
6. 生成成功后进入“文案已就绪”，可编辑标题、正文、引用、署名，也可重新生成。
7. 点击“使用这份文案 →”。
8. 选择风格 → 点击“预览效果 →”。
9. 预览接收端效果 → 可“调整风格”或“生成链接 ✦”。
10. 生成成功 → 复制链接、打开预览效果或再创建一份。

接收流程：

1. 打开链接。
2. 进入封面状态。
3. 点击“点击开启信笺”。
4. 进入正文状态。
5. 点击“点击接收我的爱心电波”。
6. 进入接收完成状态，显示接收日期和完成态文案。

## 8. 接收端三状态

封面状态：

1. 英文提示：A message for you。
2. 收信人：如“致：最亲爱的妈妈”。
3. 引导文案：在这琐碎而温热的日常里 / 有一份心意请您亲启。
4. 主按钮：点击开启信笺；开启中显示“正在开启…”。
5. 底部品牌：心意链接 · HeartLink。

正文状态：

1. 英文标签：Acknowledgment Receipt。
2. 标题：如“妈妈，谢谢您。”。
3. 正文段落。
4. 引用块。
5. 署名：如“— 您的专属宝贝”。
6. 年份与场景标识：如“2025 · 以感谢之名”。
7. 主按钮：点击接收我的爱心电波。

完成状态：

1. 保留正文内容。
2. 主按钮变为金色完成态。
3. 根据场景显示完成文案：
   感谢：这份感谢，已被好好收藏。
   祝福：祝福已送达，心意已珍藏。
   道歉：我理解了，谢谢你的勇气。
   鼓励：谢谢，我会加油的！
   小心意：收到啦！爱意增幅 +1000%。
4. 显示“已于 2025 年 6 月 17 日 接收”。
5. 显示“这份心意已被珍藏”。

## 9. 成功页范围

生成成功页仅包含：

1. 成功标题：心意已封存。
2. 说明：专属链接已生成，分享给收信人，静待亲启。
3. 专属链接展示。
4. 复制按钮。
5. 打开预览效果。
6. 再创建一份。
7. 隐私提示：拥有链接的人都可以查看，请只分享给你想分享的人。

当前 heartlink-ui-mvp 已验收 UI 基础不包含链接管理、过期设置、分享面板、二维码、访问统计或删除入口。MVP 默认链接长期有效，暂不提供删除和过期设置；后续可支持链接过期、删除或管理，但不要把“永久有效”作为用户可见承诺。

## 10. 异常状态

当前 heartlink-ui-mvp 已验收 UI 基础已经实现并需要保留的状态：

1. 信息填写为空：收信人或想说的话为空时，AI 生成按钮置灰不可用。
2. AI 生成中：正在生成文案。
3. AI 生成失败：文案生成失败，支持重新生成或手动填写文案继续。
4. 网络连接失败：支持重新生成或手动填写文案继续。
5. 复制成功：toast 显示“链接已复制到剪贴板”。
6. 复制失败：toast 显示“复制失败，请手动复制”。
7. 接收端加载中：骨架屏。
8. 链接不存在：404 页面。
9. 链接过期：过期页面。

当前 heartlink-ui-mvp 已验收 UI 基础未体现真实 empty 数据页、真实接口错误页、真实 token 查询失败逻辑或真实网络请求。

## 11. MVP 黑名单

MVP 不做：

1. 真实支付、微信支付、支付宝、Stripe、红包或收款能力。
2. 登录、注册、账号体系。
3. App、小程序。
4. 模板市场。
5. 社交广场、点赞、评论、关注。
6. 多人协作。
7. 复杂 H5 拖拽编辑器、图层、自由排版、任意字体大小调节。
8. 图片、视频、音频上传。
9. 复杂数据分析后台。
10. 接收者回复留言。
11. PDF 导出。
12. Web Crawler。
13. 向量数据库。
14. 迁移 Next.js 或重做视觉。

## 12. 验收标准

1. 可以启动当前 Vite + React 原型。
2. 首页、场景选择、信息填写、AI 生成、文案结果、风格选择、预览、生成成功页面均可从创建端流程触达。
3. 创建端必填字段为空时，AI 生成按钮不可用。
4. AI 生成中、生成失败、网络错误状态在原型中可展示。
5. AI 文案结果页支持编辑标题、正文、quote / 引用句、按钮文案、署名。
6. 风格选择只提供 4 个原型内置风格。
7. 成功页展示专属链接，并提供复制、打开预览、再创建一份。
8. 接收端覆盖封面、正文、接收完成三状态。
9. 接收端保留 loading、链接不存在、链接过期等状态组件，后续由真实业务状态触发。
10. 不引入登录、支付、模板市场、复杂编辑器等黑名单能力。

## 13. UI 反向修正记录

相对 `docs/PRD_BASELINE.md`，本文件做了以下反向修正：

1. 产品名从“暂定名”收敛为当前 UI 显示的“心意链接 / HeartLink”。

10. 明确当前 UI 来自 Figma Make 导出代码，已经过人工正式化小改并通过人工验收；当前 heartlink-ui-mvp 是 MVP v0.1 已验收 UI 基础，后续开发不得重写 UI。
