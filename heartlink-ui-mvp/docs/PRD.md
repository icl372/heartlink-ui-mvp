## 0. 当前新链路修订

心意链接的新定位是：**帮用户把已有的心意包装好，做成一个能送出去的东西。**

当前核心链路：

```text
用户交出心意 -> 系统理解心意 -> 模型加工心意 -> 包装成可发送成品 -> 生成链接发送
```

产品判断：

1. 用户不是没有心意，而是不知道怎么把心意包装成一个好看、像样、能送出去的成品。
2. 产品不负责凭空创造心意，只负责理解、整理和包装用户已经提供的真实信息。
3. 创建端输入已从“普通填表 / 生成文案”调整为“准备这份心意”。
4. 结果页不再定义为“AI 文案结果”，而是“心意成品结果”。
5. 当前仍是 MVP，不包含登录、支付、模板市场、社交广场、多种包装形式或复杂编辑器。

## 1. 产品定位

心意链接 / xygift 是一个轻量的心意成品包装工具：用户交出已经存在的心意材料，系统理解并整理这些材料，再包装成一份可预览、可复制链接、可直接送给 TA 的心意成品。接收者打开链接后，像拆开一份小礼物一样经历「封面 → 正文 → 接收完成」的仪式感流程。

核心价值：把普通文字表达包装成一个温柔、私密、适合手机打开的小礼物链接，而不是让用户制作复杂 H5、设计作品，或让 AI 凭空替用户创造心意。

## 2. 目标用户

创建者：

1. 想向父母、伴侣、朋友、同事表达感谢、祝福、道歉、鼓励或小心意的人。
2. 想把转账说明、资助说明或一段普通文字变得更有仪式感的人。
3. 不想学习复杂编辑器，只想快速生成一个可分享页面的人。

接收者：

1. 通过聊天工具收到链接的人。
2. 无需登录、无需下载 App，只需打开链接并阅读。

## 3. 用户痛点

1. 普通聊天消息缺少仪式感，容易被聊天记录淹没。
2. 红包或转账体验偏交易，难以承载“我记得你、感谢你、心疼你”的表达。
3. 传统 H5 或贺卡工具过重，模板、排版和样式选择成本高。
4. 用户真正需要的是快速生成一份“可被打开的小惊喜”，不是做复杂设计。

## 4. MVP 范围

MVP 只覆盖当前 heartlink-ui-mvp 已验收 UI 基础已经呈现的能力：

1. 创建端从首页进入，完成场景选择、准备这份心意、包装心意、风格选择、预览这份心意、生成心意链接。
2. 接收端打开链接后，完成封面、正文、接收完成三状态体验。
3. 提供原型级 loading、AI 失败、网络错误、复制成功/失败、链接不存在、链接过期状态展示。
4. 不包含真实后端、真实 AI 接口、真实 token 查询、登录、支付、模板市场或复杂编辑器。

## 5. 页面清单

创建端页面：

1. 首页：品牌、定位文案、开始创建入口、接收端预览入口。
2. 场景选择页：选择感谢、祝福、道歉、鼓励、小心意。
3. 准备这份心意页：填写“这份心意送给谁”“TA 是你的谁”“为什么想送”“有什么事想放进心意里”“最想表达的重点”“补充一句话”“想要什么感觉”。
4. 心意包装中页：展示“正在包装这份心意”的 loading 状态。
5. 心意成品结果页：展示已经包装好的心意内容，支持编辑标题、正文、引用、署名；支持重新包装。
6. 风格选择页：选择温柔信纸、复古收据、诗意卡片、简约便签。
7. 预览页：展示接收端封面和正文的组合预览，支持调整风格或生成链接。
8. 生成成功页：展示心意链接、复制心意链接、预览这份心意、再做一份、隐私提示。

接收端页面/状态：

1. 加载中：骨架屏和“正在加载心意…”。
2. 封面：收信人、引导文案、场景图标、开启按钮、`xygift · 心意链接` 标识。
3. 正文：英文标签、标题、正文、引用、署名、日期/场景标识、接收按钮。
4. 接收完成：正文保留，接收按钮变为金色完成态，展示接收时间和珍藏提示。
5. 链接不存在：404 / Not Found、说明文案、查看示例效果按钮。
6. 链接过期：Link Expired、说明文案、查看示例效果按钮。

## 6. 字段清单

创建端输入字段：

1. 这份心意送给谁：必填，对应 `recipientName`。
2. TA 是你的谁：选填，对应 `recipientRole`，选项为父母、伴侣、朋友、老师、同学、同事、其他。
3. 为什么想送这份心意：必填，对应 `occasion`，选项为生日、感谢、道歉、鼓励、想念、表白、和好、其他。
4. 有什么事，想放进这份心意里：必填，对应 `story`。
5. 你最想表达的重点：选填标签，对应 `intentTag`，选项为感谢、道歉、鼓励、想念、祝福、表白、其他。
6. 也可以补一句你最想让对方知道的话：选填，对应 `coreMessage`。
7. 想要什么感觉：必填或默认选择，对应 `tone`，选项为真诚一点、温柔一点、像日常聊天一样、不要太肉麻、有仪式感一点。
8. 署名：保留现有字段，对应 `senderName`。
9. 包装后可编辑字段：标题、正文、quote / 引用句、按钮文案、署名。
10. 风格：简约便签、柔粉信笺、雾紫诗笺、日光便签。

生成结果字段：

1. 心意链接：生产环境格式为 `https://www.xygift.cn/to/:token`；本地开发可使用当前 origin。
2. 复制状态：复制成功、复制失败。

接收端展示字段：

1. 收信人。
2. 标题。
3. 正文。
4. 引用。
5. 署名。
6. 场景。
7. 接收日期。
8. 完成态文案。

## 7. 核心流程

创建流程：

1. 首页 → 点击“开始创建心意”。
2. 场景选择 → 选择一个场景。
3. 准备这份心意 → 填写收信人、关系、送出原因、具体小事、表达重点、补充话、想要的感觉和署名。
4. 点击“帮我包装这份心意”。
5. 进入“正在包装这份心意” loading。
6. 包装成功后进入“心意已包装好”，可编辑标题、正文、引用、署名，也可重新包装。
7. 点击“使用这份心意 →”。
8. 选择风格 → 点击“预览这份心意 →”。
9. 预览接收端效果 → 可调整风格或生成心意链接。
10. 生成成功 → 复制心意链接、预览这份心意或再做一份。

接收流程：

1. 打开链接。
2. 进入封面状态。
3. 点击“打开这份心意”。
4. 进入正文状态。
5. 点击“收下心意”。
6. 进入接收完成状态，显示接收日期和完成态文案。

## 8. 接收端三状态

封面状态：

1. 英文提示：A message for you。
2. 收信人：如“给妈妈的一份小心意”。
3. 引导文案：短封面提示，如“有一份心意送给你”。
4. 主按钮：打开这份心意；开启中显示轻量 loading。
5. 底部品牌：xygift · 心意链接。

正文状态：

1. 英文标签：根据主题显示，如 A SIMPLE NOTE / A NOTE FOR YOU。
2. 标题：包装后的心意标题。
3. 正文段落。
4. 引用块。
5. 署名。
6. 年份与场景标识使用真实 `createdAt` / `occasion`，不得硬编码旧日期。
7. 主按钮：收下心意。

完成状态：

1. 保留正文内容。
2. 主按钮变为金色完成态。
3. 主文案为“已收下这份心意”。
4. 底部说明使用 `acceptedText`，默认“这份心意已被珍藏”。
5. 接收日期使用真实 `acceptedAt`，不得硬编码旧日期。

## 9. 成功页范围

生成成功页仅包含：

1. 成功标题：这份心意可以送出去了。
2. 说明：把链接发给 TA，TA 打开后就能收到这份心意。
3. 心意链接展示。
4. 复制心意链接。
5. 预览这份心意。
6. 再做一份。
7. 隐私提示：拥有链接的人都可以查看，请只分享给你想分享的人。

当前 heartlink-ui-mvp 已验收 UI 基础不包含链接管理、过期设置、分享面板、二维码、访问统计或删除入口。MVP 默认链接长期有效，暂不提供删除和过期设置；后续可支持链接过期、删除或管理，但不要把“永久有效”作为用户可见承诺。

## 10. 异常状态

当前 heartlink-ui-mvp 已验收 UI 基础已经实现并需要保留的状态：

1. 准备心意必填项为空：收信人、送出原因或具体小事为空时，包装按钮置灰不可用。
2. 心意包装中：正在包装这份心意。
3. 心意包装失败：这份心意暂时没有包装成功，支持重新包装。
4. 网络连接失败：网络好像有点不稳定，请稍后再试。
5. 复制成功：toast 显示“链接已复制到剪贴板”。
6. 复制失败：toast 显示“复制失败，请手动复制”。
7. 接收端加载中：骨架屏。
8. 链接不存在：404 页面。
9. 链接过期：过期页面。

当前 heartlink-ui-mvp 已验收 UI 基础未体现真实 empty 数据页、真实接口错误页、真实 token 查询失败逻辑或真实网络请求。

## 11. MVP 黑名单

MVP 不做：

1. 真实支付、微信支付、支付宝、Stripe、红包或收款能力。
2. 登录、注册、账号体系。
3. App、小程序。
4. 模板市场。
5. 社交广场、点赞、评论、关注。
6. 多人协作。
7. 复杂 H5 拖拽编辑器、图层、自由排版、任意字体大小调节。
8. 图片、视频、音频上传。
9. 复杂数据分析后台。
10. 接收者回复留言。
11. PDF 导出。
12. Web Crawler。
13. 向量数据库。
14. 迁移 Next.js 或重做视觉。

## 12. 验收标准

1. 可以启动当前 Vite + React 原型。
2. 首页、场景选择、准备这份心意、心意包装、心意成品结果、风格选择、预览、生成成功页面均可从创建端流程触达。
3. 创建端必填字段为空时，包装按钮不可用。
4. 心意包装中、包装失败、网络错误状态可展示。
5. 心意成品结果页支持编辑标题、正文、quote / 引用句、按钮文案、署名。
6. 风格选择只提供 4 个原型内置风格。
7. 成功页展示心意链接，并提供复制心意链接、预览这份心意、再做一份。
8. 接收端覆盖封面、正文、接收完成三状态。
9. 接收端保留 loading、链接不存在、链接过期等状态组件，后续由真实业务状态触发。
10. 不引入登录、支付、模板市场、复杂编辑器等黑名单能力。
